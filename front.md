# Frontend UI/UX Specification

## 기술 스택
- React 18 + TypeScript + Vite
- Material UI (MUI) v5
- React Router v6 (페이지 이동)
- Axios (API 호출)
- Day.js (날짜 처리)

---

## 화면 구성

### 1. 메인 페이지 (`/`)

#### 1-1. 상단 헤더 영역

```
┌─────────────────────────────────────────────────────────┐
│  < 2025년 6월 >        전체 12 | 완료 8 | 완료율 66%      │
│                        ████████████░░░░  [업무 추가 +]   │
└─────────────────────────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| `< 이전달` / `다음달 >` 버튼 | 월 이동, URL 쿼리 파라미터 `?year=2025&month=6` 반영 |
| 현재 년/월 | `YYYY년 M월` 포맷 |
| 전체 업무 수 | 해당 월 전체 Task 개수 |
| 완료 수 | `completed = true` 개수 |
| 완료율 Progress Bar | MUI `LinearProgress` + 퍼센트 텍스트 |
| `업무 추가 +` 버튼 | 클릭 시 업무 등록 Modal 오픈 |

#### 1-2. 필터 바 영역

```
┌─────────────────────────────────────────────────────────┐
│  🔍 업무 검색    [업무구분 전체 ▼]  [전체 ▼]  [오늘만 □]  │
└─────────────────────────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 검색창 | `title` 포함 검색 (프론트 필터링) |
| 업무구분 드롭다운 | 전체 / 각 Category 선택 |
| 완료 상태 드롭다운 | 전체 / 완료 / 미완료 |
| 오늘만 체크박스 | 체크 시 오늘 날짜 업무만 표시 |

#### 1-3. 업무 테이블

```
┌────────┬──────┬──────────┬────────────────────┬──────┐
│  날짜  │ 요일 │ 업무구분 │       할 일         │ 완료 │
├────────┼──────┼──────────┼────────────────────┼──────┤
│ 06/02  │  월  │  [회의]  │ 주간 팀 미팅        │  ☑  │
│ 06/02  │  월  │ [문서작성]│ 제안서 초안 작성    │  □  │
│ 06/03  │  화  │ [고객지원]│ A사 문의 처리       │  □  │
└────────┴──────┴──────────┴────────────────────┴──────┘
```

| 컬럼 | 설명 |
|------|------|
| 날짜 | `MM/DD` 포맷, 같은 날짜는 행 병합(rowSpan) |
| 요일 | 날짜 기반 자동 계산, 토=파랑, 일=빨강 |
| 업무구분 | Category `name` + `color` Chip 표시 |
| 할 일 | 완료 시 취소선 스타일 |
| 완료 체크박스 | 클릭 즉시 API 호출(낙관적 업데이트) |

- 행 우클릭 또는 행 끝 `⋮` 버튼으로 **수정 / 삭제** 메뉴 표시
- 날짜 오름차순 정렬 고정

### 2. 업무 추가/수정 Modal

```
┌─────────────────────────────────┐
│  업무 추가                    ✕ │
├─────────────────────────────────┤
│  날짜 *    [2025-06-02    📅]   │
│  업무구분 * [회의          ▼]   │
│  할 일 *   [                ]   │
├─────────────────────────────────┤
│               [취소]  [저장]    │
└─────────────────────────────────┘
```

| 필드 | 유효성 검사 |
|------|------------|
| 날짜 | 필수, MUI DatePicker |
| 업무구분 | 필수, Category 드롭다운 |
| 할 일 | 필수, 최대 200자 |

- 수정 모드: 기존 값 pre-fill, 버튼 텍스트 "수정"으로 변경

### 3. 설정 페이지 (`/settings/categories`)

```
┌─────────────────────────────────────────┐
│  설정 > 업무구분 관리      [+ 구분 추가] │
├─────────────────────────────────────────┤
│  ● 회의          파랑   ↕  ✏  🗑        │
│  ● 교육          초록   ↕  ✏  🗑        │
│  ● 고객지원      주황   ↕  ✏  🗑        │
│  ● 문서작성      보라   ↕  ✏  🗑        │
│  ● 개인          회색   ↕  ✏  🗑        │
└─────────────────────────────────────────┘
```

| 기능 | 설명 |
|------|------|
| 추가 | 이름 + 색상 선택 Modal |
| 수정 | 인라인 편집 또는 Modal |
| 삭제 | 해당 구분을 참조하는 Task가 있을 경우 경고 후 삭제 |
| 순서 | `sortOrder` 기반, 드래그 앤 드롭(추후 확장) |

## 컴포넌트 구조

```
App
├── AppLayout (Sidebar + Topbar)
│   ├── Sidebar: 홈 / 설정 네비게이션
│   └── Topbar: 앱 타이틀
├── MainPage
│   ├── MonthHeader (년월 이동 + 통계)
│   ├── FilterBar (검색 + 필터)
│   ├── TaskTable
│   │   └── TaskRow (날짜별 행)
│   └── TaskFormModal (추가/수정)
└── SettingsPage
    └── CategoryManager
        └── CategoryFormModal (추가/수정)
```

## 상태 관리

| 상태 | 위치 | 설명 |
|------|------|------|
| 현재 년/월 | URL query param | 새로고침 시 유지 |
| 필터 상태 | `useFilter` hook (로컬) | 검색어, 구분, 완료여부, 오늘만 |
| Task 목록 | `useTask` hook (서버 상태) | API 응답 캐시 |
| Category 목록 | `useCategory` hook (서버 상태) | 앱 초기 로드 시 fetch |
